Sécurité du JavaScript côté client

Le code JavaScript exécuté dans le navigateur est toujours accessible à toute personne qui sait utiliser les outils de développement (DevTools). Cela signifie que toute donnée sensible comme :
  • un mot de passe,
  • une clé API,
  • un token d’accès,
  • ou une logique de sécurité,
peut être vue ou copiée par n’importe quel utilisateur.

Méthodes pour améliorer la sécurité côté client :

  • Déplacer la logique sensible vers le backend
  • Utiliser l'obfuscation ou la minification
  • Variables d’environnement
  • Utiliser HTTPS
  • Authentification sécurisée avec des tokens (comme JWT)
Même si on peut rendre le JavaScript plus difficile à comprendre, on ne peut jamais garantir une sécurité totale côté client.
Pour toute donnée ou logique sensible, il faut utiliser le back-end.
